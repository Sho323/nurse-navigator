# AI経営・加算ナビゲーションシステム (Nurse Navigator) 実装指示プロンプト

このドキュメントは、AIコーディングアシスタント（Cursor, v0, Bolt.new等）に読み込ませて一気に土台を作らせるためのメガプロンプトです。

## 【AIへの指示内容】
あなたはシニアフルスタックエンジニアです。以下の仕様書群（`spec.md`, `test-spec.md`, `rls-design.md`）と、本プロンプトの指示に従い、Next.js + Supabase + Tailwind CSS構成の訪問看護ステーション向けWebアプリケーション「Nurse Navigator」のMVP版を構築してください。

### 1. プロジェクトの初期化と必須ライブラリ
*   Next.js (App Router), TypeScript, Tailwind CSSでプロジェクトを作成してください。
*   UIコンポーネントライブラリとして `lucide-react` (アイコン用) を導入してください（複雑なshadcn/ui等は一旦不要で、Tailwindで直接スタイリングします）。
*   Supabaseクライアント (`@supabase/supabase-js`, `@supabase/ssr`) を導入してください。
*   テストフレームワークとして `jest`, `@testing-library/react`, `@testing-library/user-event` をセットアップしてください。

### 2. デザインシステムの適用
*   アプリ全体の色調は、親しみやすく温かみのある**オレンジとアイボリー**を基調としてください。`tailwind.config.ts` に以下のテーマカラーを定義してください。
    *   `primary`: オレンジ系（例: `#F97316` またはヘルスケア向けの柔らかいオレンジ）
    *   `background`: アイボリー系（例: `#FFFAF0`）
    *   `surface`: 白（カード等の背景色）
*   角丸（`rounded-2xl` 程度）や十分な余白（`p-6`, `gap-4` 等）を多用し、業務システム特有の堅苦しさを排除したモダンな「ヘルスケア・トラッカーアプリ」風のUIを作成してください。ボタンや入力欄はスマホでのタップを意識して大きく（`min-h-[48px]` 等）してください。

### 3. モックデータの作成とUI実装
まずはSupabaseに繋ぎ込まず、フロントエンドの見た目（静的モック）から構築します。以下の5画面を作成してください。
詳細なレイアウト指示は以下です。

#### (1) ログイン画面 (`/login`)
*   白背景・角丸・シャドウ付きの中央配置カード。
*   ロゴ名「Nurse Navigator」と、大きな「Googleでログイン」ボタンのみ。
*   背景全体は薄いアイボリー。

#### (2) 訪問看護師用 ホーム/記録画面 (`/` または `/nurse`) (モバイルファースト)
*   下部にボトムタブナビゲーション（ホーム、チャット、スケジュール設定）。
*   上部に本日の訪問予定リスト（横スワイプ可能なカード形式）。
*   カードタップで記録入力モードへ。
*   入力項目: バイタル入力（大きく押しやすい数値入力）、目立つ「マイクアイコン」の音声入力付きテキストエリア、写真添付ボタン。

#### (3) 管理者ダッシュボード (`/admin/dashboard`) (PCファースト)
*   左固定サイドバー。
*   最上部に「コンパニオンアプリ（Kompanion）」風の大きく見やすいサマリー数字（今月の売上、未収金総額等）。
*   その下に売上推移グラフ（ダミーの棒グラフ等でOK）。
*   さらに下に、目立つ（薄いオレンジ背景）AI加算アラートリスト。

#### (4) CSVアップロード＆消込画面 (`/admin/reconciliation`) (PC)
*   上部に点線枠のだドラッグ＆ドロップエリア。
*   下部に消込結果データテーブル（請求額、入金額、ステータスバッジ、AIコメント）。

#### (5) 情報共有チャット (`/chat`) (レスポンシブ)
*   LINE風の双方向吹き出し。
*   AIからの通知ログは背景色を変えて目立たせる。

### 4. Supabase DB推測と型定義の作成
UIモックが完成したら、`rls-design.md` を参照の上、Supabaseで必要なテーブルのTypeScript型定義（`types/supabase.ts`）を作成してください。

### 5. ロジックの組み込み (可能な範囲で)
*   Authのログイン・ログアウト処理（Google OAuth呼び出し）。
*   Web Speech API の Reactカスタムフック化とコンポーネントへの結合。（ブラウザの制限に注意して実装すること）
*   テストコード（`test-spec.md`記載内容）を作成し、静的UI上でパスすることを確認してください。
