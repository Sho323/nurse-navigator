# AI経営・加算ナビゲーションシステム (Nurse Navigator) テスト仕様書 (TDD)

## 1. テスト方針

*   **テストフレームワーク**: Jest + React Testing Library
*   **アプローチ**: テスト駆動開発 (TDD) を前提とし、UIコンポーネントの振る舞いおよロジックを検証する。
*   **モック戦略**:
    *   外部API（OpenAI API, Google Maps API等）との通信は全てモック化（`jest.mock`）し、テストリソースを節約する。
    *   Supabaseクライアント（Auth, DBアクセス）も原則モック化し、UI/UXのロジック検証に集中する。

## 2. テスト対象とテストケース

### 2.1 認証機能 (Login / Auth)

*   **正常系**:
    *   [ ] 「Googleでログイン」ボタンをクリックすると、SupabaseのOAuthサインインメソッドが呼ばれること。
    *   [ ] ログイン成功後、ユーザーのロールに応じたトップ画面（管理者→ダッシュボード、看護師→ホーム）にリダイレクトされること。
*   **異常系**:
    *   [ ] ログインエラー時に、適切なエラーメッセージが画面に表示されること。

### 2.2 現場業務支援 (訪問記録)

*   **正常系**:
    *   [ ] バイタル情報（体温、血圧など）を入力し、保存ボタンを押すとDB保存メソッドが呼ばれること。
    *   [ ] 音声入力（Web Speech API）で取得したテキストが、記録テキストエリアに正しく反映されること。
*   **異常系**:
    *   [ ] **【重点】**写真アップロード時、ファイルサイズ制限（例: 5MB以上）を超えた場合にエラーメッセージが表示され、アップロード処理が中断されること。
    *   [ ] 必須項目（訪問種別など）が未入力の場合に、バリデーションエラーが表示されること。

### 2.3 現場業務支援 (情報共有チャット)

*   **正常系**:
    *   [ ] テキストを入力して送信ボタンを押すと、メッセージ送信メソッドが呼ばれ、入力欄がクリアされること。
    *   [ ] Supabase Realtimeからの新規メッセージイベントを受信した際、チャット画面に新着メッセージが追加表示されること。

### 2.4 経営支援 (AI自動消込・データアップロード)

*   **正常系**:
    *   [ ] 請求CSVと入金CSVを両方正しくドロップ（選択）した状態でのみ、アップロード（解析開始）ボタンが活性化すること。
    *   [ ] アップロード後、**【モック化した】**AI解析APIから成功レスポンスが返ってきた際、画面に消込結果テーブル（マッチ、要確認等のステータス）が正しく描画されること。
*   **異常系**:
    *   [ ] CSV形式以外のファイルをドロップした場合にエラーメッセージが表示されること。

### 2.5 アクセス権限 (RLS・画面表示制御)

*   **正常系**:
    *   [ ] 経営者/管理者権限を持ったユーザーでログイン時、サイドメニューに「ダッシュボード」「CSVアップロード」が表示されること。
    *   [ ] 同一事業所内の別の看護師が作成した訪問記録が、自分の画面にも表示される（相互閲覧可能）こと。
*   **異常系**:
    *   [ ] **【重点】**訪問看護師権限のユーザーでログイン時、`/dashboard` などの管理者専用URLに直接アクセスしようとした場合、何も表示されない（真っ白な画面、またはコンポーネントがレンダリングされない）設計通りにブロックされること。

## 3. テストコードの実装例 (Jest + React Testing Library)

以下は、「2.2 現場業務支援（訪問記録） - 写真のファイルサイズ超過エラー」のテストコード実装例（プロンプト用ベース）です。

```javascript
import { render, screen, fireEvent, waitFor } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import VisitRecordForm from '../components/VisitRecordForm';

describe('VisitRecordForm Component', () => {
  it('写真アップロード時、ファイルサイズが5MB以上の場合にエラーメッセージを表示し処理を中断すること', async () => {
    // 画面をレンダリング
    render(<VisitRecordForm />);

    // 6MBのダミーファイルを作成
    const largeFile = new File(['x'.repeat(6 * 1024 * 1024)], 'large_image.jpg', { type: 'image/jpeg' });
    
    // file input要素を取得（※実際のDOM構造に合わせて調整が必要）
    const fileInput = screen.getByLabelText(/写真添付/i);

    // ファイルをユーザー操作としてアップロード
    await userEvent.upload(fileInput, largeFile);

    // 送信ボタンをクリック
    const submitButton = screen.getByRole('button', { name: /保存/i });
    fireEvent.click(submitButton);

    // 期待値: DB保存処理が呼ばれず、画面上にエラーメッセージが表示されること
    await waitFor(() => {
      expect(screen.getByText(/ファイルサイズが大きすぎます/i)).toBeInTheDocument();
    });
  });
});
```
